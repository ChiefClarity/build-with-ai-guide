The Story

It Started With Pipe Cleaners

My daughter Amilia (9) and her cousins Siena (10) and Ella (7) started making pumpkins out of pipe cleaners. One day I found them making little signs and trying to sell those pumpkins out of the front window of our house as people walked by. The opportunity just hit me: let's teach them to build something with AI to market and manage their little business the modern way. So we sat down together and built them a real business, website, dashboard, and all. They named it Golden Stems, after the golden pipe cleaners they used to make their very first pumpkins.

But let me back up. A few years ago I left the corporate world and bought a small Main Street business. To modernize it, I spent the next two and a half years learning everything I possibly could about AI, and that obsession eventually became Clarity, an AI-native startup I founded for the home services industry. I'm not an engineer. I'm just a dad who refused to let this technology pass him by.

That whole journey, learning to operate a business, learning to build with AI myself, then working alongside a professional dev team to build Clarity, completely changed how I see what's coming. AI is about to confront the world with a major paradigm shift, and I believe this journey has prepared me for it. But the part that means the most to me is this: I get to pass this knowledge and these skills on to my kids. They'll grow up with AI as a normal part of life, but they won't learn how to truly use it in school. I believe it's on us, as parents, to teach them, but we have to embrace it ourselves first. And it matters more than you might think.

So I made this for the parents who feel concerned, confused, or even a little afraid of AI. I want to show you it doesn't have to be intimidating, that there's so much more to it than the free version of ChatGPT, and that even if you've never written a line of code, you can learn to use it, and teach your kids to use it too.

Here's how we did it: I used Claude Code, an AI tool you talk to in plain English, and it wrote all the code. The girls were right next to me the whole time. They chose the colors, named the business, picked the fonts, came up with the "Mystery Pack" idea, gave feedback on every page ("make that button bigger, Dad"), and watched their ideas come to life in real time.

That's the real magic. It's not about the website. It's about sitting with your kid, building something they're proud of, and learning a skill together that will shape not just their generation, but more and more of how all of us live, work, and get things done from here on.

Your kid has something like this too. Maybe it's bracelets, 3D-printed figurines, stickers, or baked goods. Whatever they make, this guide shows you how to turn it into a real business you build together.

Set Up Your Workspace

1–2 hours
↓

Before we build anything, let's get your tools ready. The good news: Claude Code does almost all of this for you, and walks you through anything it can't. You only do this once.

1. Download Claude Code: Go to claude.com/claude-code and download the desktop app. Install it like any other app and sign in with your account. No terminal, no command line, just an app you talk to. (Using Claude Code requires at least a Claude Pro plan, about $20/month.)

2. Create a GitHub account: Go to github.com and sign up (free). This is where your project is saved online. From the very start, Claude Code saves a new version here every time you make a change, like an automatic backup with a full history you can always rewind to. You don't have to learn it, Claude Code does the saving for you. Bonus: that same GitHub login works on tons of developer sites, including Supabase and Vercel (the next two you'll set up). When you sign up for those, just click "Continue with GitHub" instead of creating another account.

3. Sign up for Supabase: Go to supabase.com and create a free account (remember, you can just click "Continue with GitHub"), then click "New Project." Give it your business name. Once it's ready, open Settings → API and copy three things: your Project URL, your anon/public key, and your service_role key (keep that last one private, it has full access). Don't worry about what they are, you're just copying and pasting.

4. Create your project: In Claude Code, start a new project for your business, then give it the prompt below. Claude Code builds the entire foundation, installs everything it needs, creates all the files, and connects your project to GitHub so every change is saved automatically. If your computer is missing a free tool it needs (like Node.js, the engine that runs your site), Claude Code will tell you and walk you through installing it.

Save yourself a thousand clicks. By default, Claude Code asks for your permission before each change it makes. You can let it just work instead. In Claude Code, press Shift+Tab to switch its permission mode until it stops asking you to confirm every step (you're looking for the "auto-accept" or "bypass permissions" mode). For total beginners, we recommend this: you wouldn't know what to approve anyway, and it keeps everything moving. Since this is your own project on your own computer, it's perfectly safe.
What you tell Claude
Create a new Next.js project for [Business Name] using the App Router, TypeScript in strict mode, and Tailwind CSS. Set up a clean, professional project structure: - src/app for pages (App Router convention) - src/components for reusable UI pieces - src/lib/data for read-only database queries - src/lib/services for outside integrations (email, image processing) - src/lib/validations for Zod schemas - src/lib/supabase for the database clients (below) - src/actions for Server Actions (all data writes go here) - src/types for shared TypeScript types Install these packages: @supabase/ssr @supabase/supabase-js resend @react-email/components react-dropzone recharts lucide-react zod class-variance-authority clsx tailwind-merge sharp pg Create a .env.local file with these values (I'll fill in the keys): NEXT_PUBLIC_SUPABASE_URL=[paste your Supabase project URL here] NEXT_PUBLIC_SUPABASE_ANON_KEY=[paste your anon key here] SUPABASE_SERVICE_ROLE_KEY=[paste your service role key here] REMOVEBG_API_KEY=placeholder RESEND_API_KEY=placeholder NEXT_PUBLIC_SITE_URL=http://localhost:3000 Also create a .env.local.example file with placeholder values (no real keys) so we can share the project on GitHub safely. Set up the Supabase client configuration: - A browser client (for the storefront pages customers see) - A server client (for admin dashboard operations) - An admin client using the service role key (for system operations like reserving products during checkout, which happens before the customer is logged in) - Middleware helper for managing auth sessions Finally, set up version control: initialize git, add a .gitignore so my secrets and node_modules are never shared, and push this project to a new private GitHub repository. As we keep building together, commit and push the changes after each step so everything is backed up with a full history.

5. Add your keys: Claude Code will show you exactly where your Supabase keys go and help you drop them in. You'll add the Resend and remove.bg keys later, when we reach those steps.

6. Set the quality bar: Before we start building features, send Claude Code one more message. It sets the engineering standards for the whole project, the same standards behind Golden Stems, so everything we build is production-grade, not a rough draft. You only send this once.

What you tell Claude
Before we build anything else, set the engineering standards for this whole project, and hold to them in every step: - Use TypeScript in strict mode. Once our database exists, generate TypeScript types from the Supabase schema and use them everywhere, so the app is type-safe from the database all the way to the screen. - Validate every user input with Zod, using the same schema on the client and the server. Never trust data coming from the browser. - Keep concerns separated: read-only data fetching in src/lib/data, all data writes in Server Actions (src/actions), outside integrations in src/lib/services, and Zod schemas in src/lib/validations. - Use React Server Components to fetch data (queries run on the server, fetched in parallel when possible) and Client Components only for interactive parts like the cart and forms. - Server Actions should validate their input, return a clear { success, error } result instead of throwing, and refresh (revalidate) the affected pages after a change so the screen always matches the data. - Security first: keep all secret keys server-side (only NEXT_PUBLIC_ variables may reach the browser), rely on Supabase Row Level Security, and only use the service-role admin client for true system operations (like reserving a product during an unauthenticated checkout). - Make it reliable: handle loading and error states everywhere, and make outside services (email, image processing) non-blocking, so if one of them fails it never breaks a customer's order. - Make it accessible and mobile-first: semantic HTML, alt text on images, keyboard navigation, and strong color contrast. - After each step, run the TypeScript type checker and the linter, and fix any errors before we move on. - Remember that I'm a non-technical beginner doing this with my kid. Any time a step needs me to do something myself outside of Claude Code (like running SQL in the Supabase SQL Editor, or changing a setting on a website), walk me through it with exact, click-by-click instructions, explain any unfamiliar terms in plain English, and assume I have never done it before. Acknowledge these standards, then apply them to everything we build from here.

What you tell Claude
In our existing project, set up a production-ready Supabase database for [Business Name]. I need proper relational database design with: Tables: - Profiles: linked to Supabase Auth, with roles (admin vs. girl/creator). Only authenticated admins can manage everything else. - Auto-updating timestamps on all tables - A trigger that auto-creates a profile when a new user signs up - Seed data for default bundle pricing (3 for $9.99, 5 for $14.99, 10 for $24.99), supply cost estimates (pipe cleaner cost per product, envelope cost per order, stamp cost per order), and low inventory threshold - Set up a public storage bucket called "product-photos" for uploaded images Generate the complete SQL setup file I can paste into the Supabase SQL Editor. Include clear comments so I can understand what each section does. Add helpful indexes for the queries we'll run most (filtering products by status, finding expired reservations, listing orders by date). Once I've run the SQL, generate TypeScript types from the database schema and wire them into our Supabase clients, so the whole app is type-safe end to end.

How you actually get paid: Every order comes in marked "awaiting payment."
